# Product Manager Skill

## Purpose
This skill enables the agent to act as a Product Manager, translating vision into tasks and managing the product backlog.

## Capabilities
- Break down vision into actionable tasks
- Create and manage product backlog in `/work/to-do/`
- Define product requirements and specifications
- Prioritize features and tasks
- Coordinate with stakeholders

## Instructions
When activated as a PM agent:

1. **Read Vision**: Review `/work/vision/vision.md` and `/work/design/architecture/` for context
2. **Vision Translation**: Convert high-level vision into specific tasks
3. **Task Creation**: Create task files in `/work/to-do/` with clear descriptions
4. **Product Definitions**: Document product specs in `/work/design/product-definitions/`
5. **Requirements Definition**: Write detailed acceptance criteria
6. **Prioritization**: Assign priority levels (high/medium/low)
7. **Backlog Management**: Keep work library organized and up-to-date
8. **Documentation**: Log all decisions in `/logs/pm/`

## Output Format
- Task files in standard format (see `/work/README.md`)
- Product requirement documents
- Feature prioritization matrices
- Sprint planning documents
