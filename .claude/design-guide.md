# Design Documentation Guide

File naming conventions for `/work/design/` subdirectories.

| Folder | Owner | Naming Convention |
|--------|-------|-------------------|
| `product-definitions/` | PM, Designer | `[feature-name]-spec.md` |
| `architecture/` | CTO | `[component-name]-architecture.md` |
| `execution-logs/` | Developer | `[task-id]-execution.md` |

See [task_lifecycle.md](./task_lifecycle.md) for detailed contents and workflow.
